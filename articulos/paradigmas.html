Paradigmas de Programación + Arquitectura Cliente-Servidor

Los paradigmas Un paradigma es como una "filosofía" o un plano general que dicta cómo debes estructurar y pensar la lógica de tu código a la hora de resolver un problema. son enfoques teóricos y estilos fundamentales para el desarrollo de software. Estos se dividen en dos vertientes principales, determinadas por el grado de interacción directa que requieren con las abstracciones de bajo nivel del computador.

Paradigmas Imperativos

El enfoque imperativo implica un desglose exhaustivo y secuencial de los pasos exactos que necesita realizar el computador para producir un resultado. Se caracteriza por el uso intensivo de la asignación y mutación de variables Es la acción de cambiar el valor almacenado en un espacio de memoria durante la ejecución de un programa. Muy común en lenguajes imperativos (ej. x = x + 1). , estructuras de control de flujo (bucles y condicionales), y en lenguajes de más bajo nivel, la manipulación directa de direcciones en memoria.

Ejemplos de paradigmas imperativos:

  1. Programación Procedimental: Este modelo se fundamenta en la estructuración de instrucciones a través de procedimientos o subrutinas reutilizables, permitiendo que la ejecución del programa se desarrolle de manera estrictamente secuencial.
  2. Programación Orientada a Objetos (POO): Articula el desarrollo de software mediante entidades u objetos que amalgaman tanto el estado como el comportamiento, promoviendo niveles superiores de abstracción y encapsulamiento Un principio de la POO que agrupa los datos y los métodos que operan sobre esos datos en una sola unidad (la clase), ocultando los detalles internos de funcionamiento al exterior. .
  3. Programación Estructurada: Se orienta a perfeccionar la claridad y calidad del código empleando únicamente subrutinas y estructuras de control lógico, con el fin de prescindir de saltos de flujo incondicionales como el comando "goto".

Tecnologías y lenguajes representativos:

  • C: Es el referente clásico de la programación procedimental y estructurada, ideal para el control de bajo nivel y la gestión de memoria.
  • Java: Un lenguaje fuertemente tipado diseñado fundamentalmente alrededor de la Programación Orientada a Objetos.
  • Python: Aunque es multiparadigma, su uso más extendido y fundamental suele basarse en la programación imperativa y orientada a objetos.

Esta clase de paradigmas es indispensable cuando es necesario tener un control estricto sobre el rendimiento y la forma específica en la que el hardware ejecuta una tarea. Sin embargo, a medida que aumenta la complejidad del software, el código puramente imperativo puede volverse difícil de mantener y contraproducente si no se estructura adecuadamente.

Paradigmas Declarativos

Cuando el manejo de bajo nivel pierde relevancia frente a la necesidad de ensamblar sistemas complejos, la declaratividad se vuelve el enfoque ideal para trabajar sobre niveles más altos de abstracción. Consiste en describir qué es lo que se quiere conseguir en lugar de detallar cómo lograrlo. En este estilo, el desarrollador describe el resultado deseado de forma consistente con las reglas del lenguaje, confiando en que el motor subyacente infiera y ejecute los pasos intermedios apropiados.

Ejemplos específicos de paradigmas declarativos:

  1. Programación Lógica: El programa se expresa como un conjunto de hechos y reglas lógicas. El motor deductivo del lenguaje se encarga de encontrar las respuestas a las consultas realizadas.
  2. Programación Funcional: Trata la computación como la evaluación de funciones matemáticas, evitando cambiar el estado y mutar los datos, lo que elimina efectos secundarios Ocurre cuando una función modifica el estado de un programa fuera de su propio ámbito local (por ejemplo, alterando una variable global o escribiendo en un archivo). La programación funcional busca evitarlos. indeseados.
  3. Programación Reactiva: Se centra en los flujos de datos asíncronos y la propagación del cambio. Si un valor cambia, todos los elementos que dependen de él reaccionan y se actualizan automáticamente.

Tecnologías y lenguajes representativos:

  • SQL (Structured Query Language): Es el lenguaje declarativo por excelencia; le indicas a la base de datos qué información necesitas (ej. SELECT * FROM usuarios), sin explicarle el algoritmo de búsqueda.
  • Haskell: Un lenguaje puramente funcional donde todo es una expresión matemática y no existe la mutación de estado.
  • React (junto a JavaScript): Una biblioteca que materializa la programación reactiva y declarativa en el desarrollo web. En lugar de manipular el DOM Document Object Model. Es una representación en forma de árbol del documento HTML. Manipularlo implica cambiar la estructura, estilo o contenido de la página web dinámicamente. manualmente paso a paso, declaras cómo debe verse la interfaz para un estado dado, y la tecnología se encarga de actualizarla eficientemente.

Arquitectura Cliente-Servidor

Es una estructura de aplicación distribuida que divide las tareas o cargas de trabajo entre dos entidades con roles bien definidos:

  • El Servidor: Un proveedor de recursos, que ofrece procesamiento de datos, lógica centralizada y almacenamiento.
  • El Cliente: Un solicitante de servicios, encargado de realizar peticiones al servidor, recibir las respuestas y renderizar la respectiva visualización gráfica para el usuario.

Esta arquitectura es indispensable cuando una aplicación necesita manejar una base de datos centralizada (para evitar inconsistencias) o realizar procesamientos intensivos que sobrepasarían las capacidades del dispositivo cliente. Es el modelo estándar para aplicaciones bancarias, videojuegos multijugador o redes sociales.

En términos materiales, el cliente y el servidor suelen ser procesos que se ejecutan en computadores distintos conectados a través de una red (como Internet o una intranet). Esto hace que la conectividad sea un requisito obligatorio y un posible punto de fallo. Además, el servidor debe estar diseñado para manejar peticiones concurrentes de múltiples clientes y ser altamente tolerante a fallos.

Arquitectura de tres capas

La arquitectura de tres capas es la evolución natural del modelo cliente-servidor tradicional. Surge para resolver problemas de escalabilidad y mantenimiento, aislando las distintas responsabilidades del software en tres niveles lógicos (y frecuentemente físicos) independientes:

  1. Capa de Presentación (Frontend / Cliente): Es la interfaz de usuario. Su única responsabilidad es traducir las tareas y resultados a un formato que el usuario pueda entender e interactuar. No procesa datos complejos ni accede a la base de datos directamente.
  2. Capa de Lógica de Negocio (Aplicación / Middleware Software que se sitúa entre el sistema operativo y las aplicaciones, o entre distintas capas de una arquitectura, facilitando la comunicación y gestión de datos (como un puente o mediador). ): Es el "cerebro" del sistema. Contiene las reglas, cálculos y flujos de trabajo específicos del negocio. Procesa la información enviada por la capa de presentación y toma decisiones antes de interactuar con la siguiente capa.
  3. Capa de Datos (Backend / Base de Datos): Responsable exclusiva del almacenamiento, recuperación y persistencia de la información.

Al separar la interfaz, la lógica y los datos, los desarrolladores pueden actualizar o escalar una capa específica (por ejemplo, cambiar la base de datos o rediseñar la interfaz visual) sin afectar ni romper el funcionamiento del resto del sistema.

Sitio web

Un sitio web es un conjunto estructurado e interrelacionado de páginas web, contenido multimedia (imágenes, videos) y recursos técnicos (hojas de estilo CSS, scripts ejecutables) que comparten un mismo dominio de Internet. Estas páginas están formateadas principalmente bajo el estándar HTML y se alojan en un servidor web.

Cualquier usuario puede acceder a un sitio web a través de un navegador cliente, conectándose mediante protocolos de red estandarizados (HTTP o HTTPS) a través de la World Wide Web. Los sitios pueden ser desde documentos de texto estáticos y puramente informativos, hasta aplicaciones web dinámicas y altamente interactivas.

Creación y configuración de un sitio web

La implementación y despliegue de un ecosistema web contemporáneo se articula a través de diversas fases técnicas y operativas:

  1. Planificación y Diseño: Comprende la definición de la arquitectura de información, el diseño de la experiencia de usuario (UX) y la creación de interfaces visuales (UI) para garantizar la coherencia funcional del proyecto.
  2. Desarrollo y Programación: Representa la etapa de construcción técnica del software, estructurada en dos dimensiones fundamentales:
    • Frontend: Codificación de la capa de presentación mediante lenguajes declarativos e imperativos (HTML, CSS y React), permitiendo la interacción fluida del usuario final.
    • Backend: Creación de la lógica de negocio centralizada y gestión de la persistencia de datos mediante motores de bases de datos robustos.
  3. Adquisición de Dominio y Alojamiento (Hosting): Fase crítica para la gestión de activos digitales y la puesta en línea del sistema:
    • Dominio: Registro de un identificador alfanumérico legible (ej. tusitio.com) que los servidores DNS Domain Name System. Es el "directorio telefónico" de Internet. Traduce nombres legibles para humanos (como www.google.com) en direcciones IP legibles por máquinas. traducen unívocamente a una dirección IP de red.
    • Hosting: Configuración de la infraestructura computacional, ya sea física o virtual, encargada de mantener la disponibilidad del sitio de forma ininterrumpida.
  4. Configuración del Servidor y Despliegue:
    • Transferencia de los activos digitales desde entornos de desarrollo hacia el servidor de producción mediante protocolos de integración continua o transferencia segura.
    • Implementación de estándares de seguridad SSL/TLS para el cifrado de tráfico y la habilitación obligatoria del protocolo seguro HTTPS.
    • Definición y ajuste de los privilegios de acceso y permisos de archivos dentro del sistema operativo del servidor.
  5. Mantenimiento: Asegura la estabilidad del sistema a largo plazo mediante el monitoreo de tráfico, optimización de rendimiento, actualización de dependencias y gestión de respaldos de información.
